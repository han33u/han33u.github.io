# 트레일 러닝 콘셉트 포트폴리오 프롬프트

이 프롬프트는 포트폴리오 사이트를 “트레일 러닝을 즐기는 사용자”의 분위기에 맞게 다시 디자인하고, 내용도 자유롭게 수정할 수 있도록 돕기 위한 지시문입니다.

이 문서는 다음을 모두 포함해야 합니다.

- 시각적 디자인 방향
- 섹션별 콘텐츠 방향
- 문구 수정 규칙
- 트레일 러닝 콘셉트에 맞는 UI/UX 요소
- 필요 시 기존 문구를 임의로 재작성할 수 있는 허용 범위

## 1. 전체 목표

사이트는 단순한 개인 포트폴리오가 아니라, 트레일 러닝을 즐기는 사람의 라이프스타일과 작업 성향이 함께 드러나는 개인 웹사이트여야 합니다.

사이트가 전달해야 하는 인상은 다음과 같습니다.

- 자연스럽고 활동적인 느낌
- 산길, 러닝, 이동성, 지속성
- 차분하지만 에너지가 있는 분위기
- 기록과 루틴을 중요하게 여기는 태도
- 정적 웹사이트답게 깔끔하고 빠른 구성

기존의 일반적인 프로필형 포트폴리오에서 벗어나, 트레일 러닝의 이미지를 디자인 언어로 녹여내세요.

## 2. 디자인 콘셉트

다음 중 일부를 적극적으로 반영할 수 있습니다.

- 흙길, 산길, 숲길을 연상시키는 자연색 계열 팔레트
- 녹색, 갈색, 베이지, 안개빛 회색 같은 부드러운 색감
- 러닝 코스, 고도, 지도, 거리감, 체크포인트를 떠올리게 하는 카드형 정보 표현
- 여백이 많은 편안한 레이아웃
- 움직임과 리듬을 느끼게 하는 섹션 구성
- 모바일에서도 쉽게 읽히는 간결한 타이포그래피
- 정보 카드에 작은 아이콘, 배지, 라벨, 상태 태그를 넣는 방식

원하면 다음 요소도 넣을 수 있습니다.

- 트레일 맵 느낌의 라인
- 등고선 패턴
- 고도 그래프 느낌의 장식
- 러닝 로그 카드
- 주행 거리나 루틴을 보여주는 요약 박스

단, 장식이 너무 강해져서 내용 읽기가 어려워지면 안 됩니다.
디자인은 콘셉트를 돕는 수준에서만 사용해야 합니다.

## 3. 콘텐츠 수정 허용 범위

이 프롬프트에서는 내용을 “임의로 수정할 수 있음”을 허용합니다.

즉, 다음을 자유롭게 조정할 수 있습니다.

- 섹션 제목
- 자기소개 문장
- 프로젝트 설명
- 경험 설명
- 연구 또는 기록 섹션의 문구
- 연락처 안내 문구
- 게임 소개 문구
- CTA 문구

다만 완전히 무작위로 바꾸는 것이 아니라, 트레일 러닝이라는 콘셉트와 자연스럽게 이어져야 합니다.

예를 들어 다음과 같은 방향은 허용됩니다.

- “정적 웹사이트” → “러닝 로그처럼 가볍고 빠르게 읽히는 웹사이트”
- “프로젝트 소개” → “러닝 루틴, 기록, 기술 실험을 함께 보여주는 카드”
- “경력 소개” → “지속성, 자기관리, 반복 개선을 중요하게 여기는 태도”

원하면 개인 정보가 일부 비어 있어도 콘셉트 중심으로 문구를 재구성할 수 있습니다.

## 4. 톤과 문체

문체는 다음과 같이 유지하세요.

- 간결함
- 자신감
- 부드러움
- 자연친화적 느낌
- 너무 무겁지 않은 전문성

문장은 길지 않게 쓰되, 어색하게 끊기지 않도록 다듬으세요.

트레일 러닝을 즐기는 사람의 느낌이 다음처럼 드러나면 좋습니다.

- 꾸준히 달리는 사람
- 기록을 남기는 사람
- 장거리 루틴을 중요하게 생각하는 사람
- 자연 속에서 리듬을 찾는 사람

## 5. 섹션별 작성 지침

### Hero

Hero는 첫 화면에서 콘셉트를 바로 전달해야 합니다.

권장 내용:

- 트레일 러닝 감성
- 정적 포트폴리오
- 개인의 리듬과 기술 작업
- 짧은 소개 문장

예시 방향:

- “트레일 러닝의 리듬처럼 차분하게 쌓아가는 개인 포트폴리오”
- “자연과 기록, 그리고 정적 웹사이트를 함께 담은 공간”
- “러닝 루틴처럼 꾸준히 완성해 가는 프로필 사이트”

Hero에는 다음 요소를 넣을 수 있습니다.

- 거리
- 루틴
- 기록
- 산길
- 체크포인트

### About / Profile

About 섹션은 사용자의 성향을 설명하는 곳입니다.

트레일 러닝을 즐기는 사람이라는 점이 자연스럽게 드러나야 합니다.

예시 방향:

- “산길과 러닝을 즐기며, 꾸준한 루틴과 기록을 중요하게 생각하는 사람”
- “트레일 러닝처럼 길고 꾸준한 흐름을 선호하는 웹 제작자”
- “움직임과 정리를 함께 좋아하는 개인 프로필”

이 섹션은 다음을 함께 담을 수 있습니다.

- 러닝 습관
- 기록 방식
- 성향
- 작업 태도

### Projects

Projects 섹션은 단순한 기술 목록보다, 콘셉트가 느껴지는 카드형 구성으로 작성하세요.

예시 주제:

- 러닝 로그 웹페이지
- 트레일 코스 기록 카드
- 정적 사이트 실험
- 게임 섹션
- 반응형 UI 실험

프로젝트 설명은 다음처럼 바꿀 수 있습니다.

- “러닝 기록을 정리하는 카드형 정적 사이트”
- “트레일 코스와 메모를 함께 저장하는 개인 페이지”
- “가볍고 빠르게 열리는 모바일 대응 포트폴리오”

프로젝트 이름도 콘셉트에 맞게 수정할 수 있습니다.

예시:

- Trail Log
- Summit Notes
- Pace Archive
- Ridge Portfolio

### Experience

Experience 섹션은 기술 경력만 적는 대신, 사용자의 태도와 습관을 함께 보여줄 수 있습니다.

트레일 러닝 컨셉과 어울리는 키워드:

- 꾸준함
- 루틴
- 회복과 페이스 조절
- 반복 개선
- 장거리 집중력

예시 방향:

- “반복 개선과 리듬 유지에 강한 작업 방식”
- “정적 사이트를 작고 빠른 단위로 완성하는 습관”
- “러닝 훈련처럼 꾸준하게 기록하고 조정하는 태도”

### Research

Research 섹션은 실험 기록, 루틴 관찰, 디자인 테스트, 러닝 로그 등의 성격으로 바꿀 수 있습니다.

예시 방향:

- 러닝 기록과 UI 실험을 함께 보여주는 영역
- 산책, 러닝, 웹 제작에서 얻은 인사이트를 적는 공간
- 새로운 카드 레이아웃과 컬러 팔레트를 시험하는 섹션

원한다면 이 섹션을 다음처럼 바꿔도 됩니다.

- “최근 트레일 러닝 코스 정리”
- “러닝 리듬과 웹 레이아웃 실험”
- “고도감과 움직임이 느껴지는 인터페이스 테스트”

### Contact

Contact는 확인된 정보만 보여주되, 콘셉트에 맞는 짧은 문구를 붙일 수 있습니다.

예시 방향:

- “기록과 작업에 대한 문의는 확인된 연락처로만 받습니다.”
- “러닝과 웹 작업에 관한 협업 제안은 검증된 채널로 남겨 주세요.”
- “추가 연락 정보는 필요 시 업데이트됩니다.”

### Games

Games 섹션은 트레일 러닝 콘셉트와 연결해도 좋습니다.

예시 방향:

- “짧은 휴식 중 즐길 수 있는 미니 게임”
- “러닝 루틴 사이에 가볍게 즐기는 인터랙션”
- “지렁이 게임을 통해 움직임과 반응성을 살린 섹션”

게임 소개 문구는 너무 길지 않게, 사용자가 바로 이해할 수 있게 쓰세요.

## 6. 시각 요소를 문구와 함께 바꾸는 방법

프롬프트는 문구뿐 아니라 디자인도 함께 바꿀 수 있어야 합니다.

필요하면 아래 요소를 함께 제안하세요.

- 배경색을 자연색 계열로 변경
- 카드 모서리를 더 부드럽게 조정
- 섹션 구분선을 등고선 느낌으로 바꾸기
- 요약 카드에 거리, 시간, 루틴 같은 태그 넣기
- 히어로 영역에 러닝/산길 감성의 짧은 카피 넣기
- 게임 섹션에 “코스 체크포인트” 같은 라벨 적용

단, 디자인이 과해져서 정적 사이트의 가독성을 해치면 안 됩니다.

## 7. 임의 수정 규칙

내용을 임의로 수정할 수 있도록 하되, 다음 기준은 지키세요.

- 콘셉트와 무관한 방향으로 벗어나지 말 것
- 과도하게 소설처럼 쓰지 말 것
- 개인 정보는 필요한 만큼만 다룰 것
- 실제 배포용 사이트처럼 읽히게 할 것
- 모바일에서 너무 길어지지 않게 할 것

즉, “자유롭게 바꿀 수 있다”는 뜻은 “트레일 러닝 콘셉트에 맞게 적극적으로 재작성 가능하다”는 의미이지, 아무 내용이나 넣어도 된다는 뜻은 아닙니다.

## 8. 실제로 바로 사용할 수 있는 프롬프트

아래 문장을 그대로 사용해도 됩니다.

> 트레일 러닝을 즐기는 사용자의 개인 포트폴리오 사이트를 새롭게 작성하세요. 사이트 전체는 산길, 루틴, 기록, 지속성, 이동성 같은 이미지를 자연스럽게 느끼게 해야 합니다. 배경색, 카드 스타일, 섹션 구성, 라벨, 아이콘, 카피까지 함께 바꿔도 좋습니다. Hero, About, Projects, Experience, Research, Contact, Games 섹션을 모두 트레일 러닝 콘셉트에 맞게 재구성하세요. 내용도 필요하면 자유롭게 다시 써도 되지만, 전체적으로는 정적 포트폴리오로 읽히고 모바일에서도 읽기 쉬워야 합니다. 게임 섹션은 가볍게 즐길 수 있는 지렁이 게임과 연결하고, 트레일 러닝의 느낌을 해치지 않도록 자연스럽게 통합하세요.

문구나 디자인에 다음 키워드를 섞으면 콘셉트가 잘 살아납니다.

- 트레일
- 러닝
- 산길
- 루틴
- 기록
- 페이스
- 고도
- 체크포인트
- 회복
- 지속성
- 자연
- 이동성
- 리듬

## 10. 마무리 점검

최종 문구를 만들기 전에 아래를 확인하세요.

- 트레일 러닝 분위기가 느껴지는가
- 내용이 너무 일반적인 포트폴리오처럼 보이지 않는가
- 문구와 디자인이 서로 어울리는가
- 임의 수정이 콘셉트 안에서 이루어졌는가
- 모바일에서 여전히 읽기 쉬운가

필요하면 이 프롬프트를 바탕으로 실제 사이트 문구와 디자인까지 바로 적용할 수 있습니다.
