# Content Prompt for the Portfolio Site

Write the visible copy for the GitHub Pages portfolio site using the following rules:

- Keep the tone professional, concise, and friendly.
- Do not invent personal information, career history, education, or projects.
- For any unverified personal detail, keep the text as `[사람 확인 필요]`.
- Keep each section short enough to fit a clean responsive layout.
- Prefer concrete sentences over vague filler.
- Keep the Games section aligned with the snake game and its controls.

## Section-by-section prompt

### Hero

Introduce the site as a static personal portfolio built for GitHub Pages.
Mention that it combines a professional profile with a small game area.
Use one strong headline and one short supporting sentence.

### About

Write a brief profile summary.
If the user's name, role, or background is not confirmed, keep those parts marked as `[사람 확인 필요]`.
Make clear that only verified information should be added here.

### Projects

Describe 2 to 3 representative projects.
If the project list is not confirmed, use neutral placeholder descriptions and avoid naming imaginary work.
Each project card should have:

- a short project title
- one sentence describing what it is
- one sentence describing why it matters

### Experience

Summarize the user's development experience in a professional, neutral way.
Focus on static websites, responsive UI work, and loop-based delivery if that matches the confirmed background.
Avoid pretending specific employers or roles unless they are confirmed.

### Research

Use this section for learning, experimentation, or ongoing work.
If no research details are confirmed, say that the section will be updated after verification.

### Contact

List only confirmed contact methods and links.
If no contact details are confirmed yet, keep them marked as `[사람 확인 필요]`.

### Games

Describe the snake game as a small interactive feature inside the portfolio.
Mention keyboard controls, mobile touch support, score tracking, and restart behavior.
Keep the copy short and easy to scan.

## Final copy check

- No invented facts
- No private data exposure
- No long paragraphs
- No mismatch with the current site layout
- Keep placeholders obvious where confirmation is still needed
